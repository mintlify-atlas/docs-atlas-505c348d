bRPC is an industrial-grade RPC framework written in C++, designed for high-performance systems at massive scale. Used throughout Baidu with over 1,000,000 server instances, bRPC supports multiple protocols on a single port, user-space threading via bthread, and comprehensive built-in observability.

## Why bRPC?

<AccordionGroup>
  <Accordion title="Multi-protocol on a single port">
    bRPC servers accept baidu_std, HTTP/1.1, HTTP/2, gRPC, Redis, Memcached, Thrift, RTMP, and more — all on the same listening port. The protocol is auto-detected per connection, simplifying deployment.
  </Accordion>
  <Accordion title="High-throughput, low-latency I/O">
    Reading from different file descriptors is fully parallelized. Writing to a single fd from multiple threads uses a wait-free queue. A single fd can handle 5,000,000 16-byte messages per second under heavy contention.
  </Accordion>
  <Accordion title="User-space threading with bthread">
    bRPC creates one bthread per request and ends it when done, automatically scaling concurrency with load. bthreads are M:N multiplexed over kernel threads, giving you the coding simplicity of synchronous code without blocking OS threads.
  </Accordion>
  <Accordion title="Built-in observability">
    Every bRPC server exposes `/status`, `/vars`, `/connections`, `/flags`, `/rpcz`, and on-demand CPU, heap, and contention profilers — accessible from any browser or `curl`.
  </Accordion>
</AccordionGroup>
